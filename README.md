## 🔧 Feature Engineering

Four feature groups are engineered from raw data and merged into a single `master_dataset`:

**Account Features** (from `Cust_Account.csv`)
- `num_accounts`, `total_high_credit`, `total_balance`
- `max_past_due`, `total_past_due`, `total_payment_amt`, `num_closed_accounts`

**Delinquency Features** (parsed from `paymenthistory1` string)
- `total_delinquency` — count of `XXX`, `90`, `120` codes
- `max_delinquency` — worst single account score

**Enquiry Features** (from `Cust_enquiry.csv`)
- `total_enquiries`, `total_enq_amount`, `max_enq_amount`
- `avg_enq_amount`, `unique_enq_purpose`, `days_since_last_enquiry`

**Demographics Derived** (from `Cust_demographics.csv`)
- `customer_age` — calculated from DOB (`feature_24`)
- `income_to_credit_ratio` — income / (credit limit + 1)
- `is_metro_city` — flag for Mumbai, Bengaluru, Delhi, Pune, Gurgaon
- `credit_score` — cleaned numeric from `feature_3`

> **Total features after engineering:** 90 input columns → 85 after imputation alignment

---

## 🧠 Key Risk Drivers

Based on XGBoost feature importance:

1. `total_delinquency` — most predictive signal
2. `max_past_due` — severity of worst missed payment
3. `credit_score` — bureau score at time of application
4. `total_enquiries` — number of credit bureau pulls
5. `income_to_credit_ratio` — affordability indicator

## 🖥️ Streamlit Web App (`app.py`)

The app lets anyone run predictions without touching the notebook.

**How it works:**
1. Upload `Cust_demographics.csv`, `Cust_Account.csv`, `Cust_enquiry.csv`
2. The app runs the exact same feature engineering pipeline
3. Results show per-customer `Default_Probability` + `Risk_Category`
4. Filter, view, and download results as CSV

**Features:**
- 🌑 Full dark theme UI
- 📊 Dataset summary metrics on upload
- 🔴🟢 Colour-coded HIGH RISK / LOW RISK labels
- 🔍 Filter by risk level and probability threshold
- ⬇️ Download predictions as CSV
- ✅ Web-compatible file paths (works on Streamlit Cloud, Hugging Face, etc.)

## 🚀 How to Run Locally

### 1. Clone the repository
```bash
git clone https://github.com/your-username/credit-risk-project.git
cd credit-risk-project
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Train the model (run the notebook)
Open `Credit_Risk_Project.ipynb` in Jupyter and run all cells. This will generate the `models/` folder with all 4 `.pkl` artefacts.

### 4. Launch the Streamlit app
```bash
streamlit run app.py
```

Then open `http://localhost:8501` in your browser.
